## 2026-05-01 — Feature 2: Engine-root inference helper — DONE

Added `crates/libaipm/src/discovery/source.rs` with `pub fn infer_engine_root(path: &Path, project_root: &Path) -> Option<(Engine, PathBuf)>`.

Behavior:
- Walks path.ancestors() upward until project_root is reached.
- Returns the FIRST ancestor whose file_name matches `.claude` (Claude), `.github` (Copilot), or `.ai` (Ai). Innermost wins, mirroring the unified-discovery design where each feature is associated with the engine that authored it, not the marketplace host that contains it.

- Returns None if path == project_root, or if no engine ancestor exists.

Re-exported from `discovery/mod.rs` as `pub use source::infer_engine_root`.

## 2026-05-01 — Feature 6: Top-level classifier dispatch — DONE

Added `crates/libaipm/src/discovery/classify.rs` with `pub fn classify(path: &Path, project_root: &Path, _fs: &dyn Fs) -> Option<DiscoveredFeature>`.

Algorithm:
1. Extract file_name from path; None → return None.
2. Call `source::infer_engine_root(path, project_root)`; None → return None (no engine ancestor).
3. Try `instruction::classify(file_name, path, engine, source_root)` first — wins over the agent rule so that `agents/AGENTS.md` is classified as Instructions, not Agent (preserves today's classify_feature_kind precedence comment at discovery_legacy.rs:259-260).
4. Match on file_name:
   - `SKILL.md` → match_skill
   - `hooks.json` → match_hook
   - `aipm.toml` → match_plugin
   - `marketplace.json` → match_marketplace
   - `plugin.json` → match_plugin_json
   - `*.md` (case-insensitive via `path.extension().is_some_and(|ext| ext.eq_ignore_ascii_case("md"))`) → match_agent
   - else → None

The `_fs` parameter is reserved per spec for matchers that may need filesystem validation in later spec features. Today's matchers are path-only.

Re-exported as `pub use classify::classify as classify_path` from `discovery/mod.rs`.
