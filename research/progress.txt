## 2026-03-17 — Session 3: Semver Parsing & Version Ranges

### Completed Features

**Feature 7: P0 — Semver parsing, version ranges, and pre-release handling** ✅
- Created `crates/libaipm/src/version.rs` module with `Version`, `Requirement`, and `Error` types wrapping the `semver` crate
- `Version::parse()` validates MAJOR.MINOR.PATCH format with optional pre-release and build metadata
- `Requirement::parse()` supports caret (`^1.0`), tilde (`~1.0`), exact (`=1.0.0`), wildcard (`*`), and compound ranges (`>=1.0, <2.0`)
- `Requirement::matches()` checks if a version satisfies the requirement
- `Requirement::select_best()` picks the highest matching non-prerelease version from candidates
- Pre-1.0 caret ranges treat minor as breaking (`^0.2.3` = `>=0.2.3, <0.3.0`) — semver crate handles this natively
- 16 unit tests covering parsing, matching, ordering, pre-release exclusion, and candidate selection
- 7 BDD step implementations added to `bdd.rs` for versioning scenarios
- All 5 BDD scenarios (17 scenario instances, ~45 steps) in `versioning.feature` pass ✔
- 81 total unit tests pass; `cargo clippy --workspace --tests -- -D warnings` passes
- Fixed pre-existing `redundant_clone` clippy warning in `workspace_init.rs`

### Design Decisions
- Named wrapper `Requirement` instead of `VersionReq` to avoid `module_name_repetitions` lint (module is `version`)
- Used `select_best()` to handle BDD scenarios 4-5 (pre-release exclusion and pre-1.0 caret) without a full resolver
- BDD registry simulation uses simple `Vec<Version>` — full registry client is a later feature
- Test helpers `assert_matches`/`assert_no_match` avoid verbose `unwrap_or_else(|| unreachable!())` chains
